# 長照個案派案管理系統 - 使用說明文件

本系統專為台灣長照 A 單位（社區整合型服務中心）的個管員與督導設計，旨在解決長照派案過程中的「時效追蹤」、「單位公平輪派」以及「派案交接」等痛點。系統具備自動化時效展延計算、BA 碼單位自動排序輪派機制，並整合 Gemini AI 提供智慧撰寫與交接輔助。

---

## 核心功能模組說明

### 1. 個案建立與時效管控
個管員建立新個案時，系統會依據「照專計畫通過日/起日」自動推算派案的完成期限：
*   **午前通過（< 12:00）**：派案完成時效為 `+2 工作日` 的中午 12:00。
*   **午後通過（>= 12:00）**：派案完成時效為 `+3 工作日` 的中午 12:00。
*   **工作日計算規則**：自動跳過台灣國定假日、例假日（週六、週日），並將政府公告的補班日計入工作日。
*   **時效逾期攔截**：當個案的「計畫送審/實際完成日」晚於系統計算出來的完成期限時，系統會自動判定為「逾時效 / 已逾時」，此時系統會強制要求個管員必須填寫「時效逾時說明」，否則無法儲存。

### 2. 公平輪排派案機制
為確保資源分配的公平與透明，系統在選擇指派 B 單位時會動態計算輪排權重：
*   **自動篩選**：系統會自動根據該個案所需的服務內容（如：BA 碼或 D 碼）來過濾顯示的 B 單位，且被標記為「停派」的單位不會出現在下拉選單中。
*   **排序梯隊**：
    1.  **首發梯隊**：歷史派案成功次數為 `0` 的單位具有最高優先權，排在選單最上方。
    2.  **輪值梯隊**：成功次數大於 `0` 的單位，將依據其「最後接案時間」由舊到新（升冪）排序。
    3.  **停派梯隊**：完全不顯示或鎖定於底部。
*   **重複派案二次確認**：若個管員指派的 B 單位近期已有成功派案紀錄，系統會在儲存時彈出「近期重複派案提醒」防呆視窗，要求二次確認以避免過度集中派案。

### 3. AI 智能輔助撰寫
系統整合 Google Gemini API，提供以下智慧輔助：
*   **逾時說明潤飾**：個管員可輸入白話的逾期理由（例如：「案家沒接電話」），點擊「AI 潤飾」後，AI 會自動將其轉化為 50 - 100 字內符合正式公文與報告規範的專業語句。
*   **一鍵生成交接短訊**：個案儲存後，個管員可一鍵生成格式化且精確的派案交接簡訊（包含案號、案主姓名、性別、服務項目等），並複製到剪貼簿中。
*   **優雅降級**：若未設定 Gemini API 金鑰或網路連線失敗，系統將自動啟動內建的降級範本（Fallback 模式），確保功能不中斷。

### 4. 案件狀態與列表管理
系統介面頂部提供三個主要分頁視圖：
*   **個案（Active Cases）**：呈現目前正在處理中、未結案的活跃案件。個管員可在此進行搜尋、編輯個案或執行結案。
*   **結案（Closed Cases）**：呈現已歸檔的個案。已結案個案將自動鎖定為唯讀狀態，僅供查詢。
*   **單位（B-Units）**：呈現所有 B 單位的詳細數據（包括成功次數、最後接案時間、目前狀態與順位），讓個管員對派案狀況一目了然。
*   **全域關鍵字模糊搜尋**：在頁面上方的搜尋列輸入「案號」、「姓名」或「督導姓名」，系統會立即以低於 500ms 的回應時間即時篩選對應案件。
